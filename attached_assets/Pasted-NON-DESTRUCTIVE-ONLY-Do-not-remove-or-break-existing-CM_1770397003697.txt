NON-DESTRUCTIVE ONLY. Do not remove or break existing CMS builder, public pages, or APIs. Prefer new files and small additions.

GOAL:
Improve the Admin CMS Page Builder experience with:
1) Preview Mode (desktop/tablet/mobile)
2) Draft vs Published preview on public site using a secure preview token
3) Inline editing for common block fields (headline, subheadline, rich text) directly in canvas
4) Block validation + warnings (missing required props, broken links, missing images/alt)
5) A “Content Checklist” panel for SEO + accessibility basics

A) ADMIN BUILDER UX IMPROVEMENTS
On /admin/cms/pages/:id:
- Add a top builder toolbar with:
  - Preview device switcher: Desktop / Tablet / Mobile
  - Preview button: opens the public page in preview mode (draft if requested)
  - Save status indicator (Saved / Unsaved changes)
  - “Validate Page” button

Canvas:
- Enable inline editing for:
  - hero.headline, hero.subheadline
  - cta_band.headline, cta_band.ctaText
  - rich_text.content (open editor modal or inline TipTap)
- Inline editing must update local block state and persist with Save

Validation:
- For each block, implement validation rules based on block schema:
  - required fields present
  - image blocks must have imageUrl + imageAlt
  - link fields must be valid path or URL
- Show block-level warnings (non-blocking) and a page-level summary panel.

Checklist panel (right side):
- Shows:
  - Title length check
  - Meta description length check
  - Missing OG image warning
  - Missing H1/headline warning (if no hero/rich_text header)
  - Missing internal CTA to /quote warning
  - Accessibility: missing alt text count
  - Link count and broken link check (simple: invalid format)

B) SECURE PREVIEW MODE
Backend:
- Add endpoint: POST /api/admin/cms/pages/:id/preview-token
  - returns a short-lived token (JWT or random stored in memory/DB)
  - token expires in 15 minutes
- Add endpoint: GET /api/public/pages/:slug?previewToken=...
  - if previewToken valid, return draft page
  - else return only published pages

Frontend public renderer:
- Accept previewToken and render draft content.
- Ensure previewToken does not expose admin APIs.

C) DOCS CATEGORIZATION UPGRADE
Admin Docs system:
- Evolve docs format to support categories and navigation:
  - Categories: CMS, SEO, Security, Integrations, Performance, CRM, Media, Theming, Routing
- Add a left-side Docs navigation with expandable categories.
- Store docs entries as structured objects:
  { category, title, bodyMarkdown, updatedAt, tags[] }

D) REQUIRED DOCS UPDATE (MANDATORY)
At the end, update Admin → Docs with a new category section:

Category: CMS
- “Builder UX Enhancements”
- “Preview Tokens (Draft Preview)”

Category: Security
- “Preview Token Security Model”

Include:
- API endpoint docs (methods, paths, auth, sample requests/responses)
- Data model changes (if any)
- How preview is implemented and expiration rules
- Validation rules list

FINAL CHECKS:
- Admin builder still saves blocks
- Preview token works and expires
- Published pages still work without token
- Docs navigation is usable and well organized
